    An individual adaptive evolution and regional collaboration based evolutionary algorithm for large-scale constrained multiobjective optimization problems

    Swarm and Evolutionary Computation 2025

    通过根据个体的进化状态向其分配计算资源来优化与收敛相关和多样性相关的变量。平衡了高维搜索空间中的收敛性和多样性

    一、问题

    1.怎样依据进化状态分配计算资源？ 2.具体如何平衡收敛性和多样性？与计算资源的关系？

    二、主要策略

    1.大规模变量被解耦为收敛相关和多样性相关的组，降低了优化复杂性，提高了收敛性和多样性。 2.实施变量分组后，根据不同个体的进化状态合理分配资源，以节省资源，提高算法的求解效率. 3.区域协作机制将辅助种群划分为子种群以探索局部区域，而主要种群则进行全局搜索。两个群体之间的合作可以保持可行性和多样性。

    三、流程

    其实就是先生成种群，进化一段时间后，对变量进行分组，然后两阶段搜索： 1.看作无约束，同时为了节省资源开销，个体15次不更新则不继续探索 2.协同进化，全局探索

    五、阶段一

    忽略约束，快速覆盖整个 UPF 并均匀分布 收敛变量优先优化 → 加速逼近 UPF。 多样性变量后续优化 → 提升分布均匀性。 个体停滞检测 → 动态节省计算资源。

    六、阶段二

    第二阶段（Stage 2）的进化机制是 “区域协作进化”（Regional Collaboration），其核心是通过 双种群协同搜索 和 子区域分工，引导种群从无约束帕累托前沿（UPF）迁移至约束帕累托前沿（CPF），同时平衡可行性与多样性。具体流程如下：

    6.1. 整体框架

    • 输入：第一阶段结束时的种群 (P)。
    • 双种群构建（Algorithm 5）：
    • 主种群（(MP)）：复制 (P) → 专注可行性（保留可行解，向 CPF 收敛）。
    • 辅助种群（(AP)）：
      1. 计算理想点 (z)（各目标最小值）。
      2. 生成 (\tau = NP/\lambda) 个参考向量（(\lambda=3)）。
      3. 将 (P) 划分为 (\tau) 个子群 (SubAP_i)，每个子群关联一个参考向量（选择与向量夹角最小的 (\lambda) 个个体）。
    • 目标：
    • (MP) 直接逼近 CPF（提升可行性）。
    • (AP) 的子群分区域探索 UPF 与 CPF 间的子空间（维持多样性）。

    6.2. 进化流程（Algorithm 6）

    (1) 子代生成
    1. (MP) 进化：
    2. 用 SBX/PM 生成 (NP) 个子代 (OMP)。
    3. (AP) 进化：
    4. 每个子群 (SubAP_i) 独立生成 (\lambda) 个子代 (SubOAP_i)（邻域交配）。
    (2) 协作重组
    1. 合并所有子代：(Oall = OMP \cup {SubOAP_1, \dots, SubOAP_\tau})。
    2. 重分配机制：
    3. 将 (Oall) 中的解重新分配到各参考向量的子区域（计算解与向量的夹角）。
    4. 目的：解决子代可能跨区域的问题（图 3a 虚线圆示例）。
    (3) 环境选择
    1. (MP) 选择：
    2. 采用 约束支配原则（CDP）[34] → 优先保留可行解，加速收敛到 CPF。
    3. (AP) 选择：
    4. 各子群独立使用 (\epsilon)-CDP[37] → 保留部分有潜力的不可行解，避免早熟收敛。

    6.3. 关键设计剖析

    组件 作用 创新点
    子区域划分 (AP) 被划分为 (\tau) 个子群，每个负责局部搜索 → 覆盖离散可行域。 解决高维空间中 CPF 碎片化问题（图 12d）。
    (\epsilon)-CDP 在 (AP) 中放宽可行性要求 → 利用不可行解探索潜在可行区域。 避免错过复杂约束边界（对比传统 CDP）。
    重分配机制 子代按参考向量重新归类 → 确保子群在目标空间定位准确。 维持子区域搜索的稳定性（图 3b）。
    双种群协作 (MP) 与 (AP) 共享子代 → (MP) 接收 (AP) 发现的优质解。 可行性 + 多样性同步提升（表 5 验证）。

    6.5. 为什么有效？

    • 可行性提升：(MP) 的 CDP 选择确保种群快速进入可行域（FR 接近 100%，图 4）。
    • 多样性维持：(AP) 的子区域搜索避免遗漏离散 CPF（对比 CMOCSO 仅找到局部可行域，图 5）。
    • 资源高效利用：Stage 1 节省的资源被用于 (AP) 的精细搜索（解决高维约束空间探索难题）。
